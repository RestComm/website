sf-flow-api

Test tool

This tutorial use SIPp. Download and install from below.

  • uas.xml
  • uac.xml

Deploy the B2BUA application

Change the sipflow-servlet.xml for your environment. The targetHost is the host address of UAS, and targetPort is the port number of UAS.

Type and run following command to build the application.

$ mvn package

Deploy the application in your environment.

Demonstration

Place uas.xml the directory that installed SIPp.

Change directory to the directory, and type following command to start as UAS.

$ sipp -i <local_address1> -p <local_port1> -sf uas.xml
Value Description
local_address1 Address of UAS (ex.127.0.0.1)
local_port1 Port number of UAS（ex.5065）

Place uac.xml the directory that installed SIPp.

Change directory to the directory, and type following command to start as UAC.

$ sipp <target_address> -i <local_address2> -p <local_port2> -sf uac.xml -m 1
Value Description
target_address Listen address of B2BUA application（ex.127.0.0.1:5060）
local_address2 Address of UAC（ex.127.0.0.1）
local_port2 Port number of UAC（ex.5055）

It is successful if Successful Call is "1" in screen of UAC.

Fig.9 Screen of UAC

You can get sources of the sample with the following command.

 $ svn co https://mobicents.googlecode.com/svn/trunk/frameworks/sip-servlets/ssf/sf-examples/sf-flow-tutorial
