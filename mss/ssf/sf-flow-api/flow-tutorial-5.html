sf-flow-api

テストツール

動作確認用にSIPpのスクリプトを用意しています。以下からダウンロードしてください。

  • uas.xml
  • uac.xml

SIPpは以下のサイトからダウンロードしてインストールしておいてください。

http://sipp.sourceforge.net/

B2BUAアプリケーションのデプロイ

sipflow-servlet.xmlの以下の部分を適時編集してください。 targetHostはUASのアドレスで、targetPortはUASのポートを指定します。

編集が終わったら以下のコマンドを実行してビルドしてください。

$ mvn package

ビルドが終わったらアプリケーションをデプロイしてください。

動作確認

UASとして動作させるマシン上でSIPpがインストールされているディレクトリに、ダウンロードしたスクリプトを配置してください。

スクリプトを配置したら、SIPpがインストールされているディレクトリに移動し、以下のコマンドを実行してください。

$ sipp -i <local_address1> -p <local_port1> -sf uas.xml
Value Description
local_address1 UASとして動作させるマシンのアドレス（例：127.0.0.1）
local_port1 UASがリスンするポート番号（例：5065）

次に、 UACとして動作させるマシン上でSIPpがインストールされているディレクトリに、ダウンロードしたスクリプトを配置してください。

スクリプトを配置したら、SIPpがインストールされているディレクトリに移動し、以下のコマンドを実行してください。

$ sipp <target_address> -i <local_address2> -p <local_port2> -sf uac.xml -m 1
Value Description
target_address B2BUAアプリケーションがリスンするアドレス（例：127.0.0.1:5060）
local_address2 UACとして動作させるマシンのアドレス（例：127.0.0.1）
local_port2 UACがリスンするポート番号（例：5055）

UACの画面で、Successful Callが1になっていたら成功です。
